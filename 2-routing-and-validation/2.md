# Big task

# Базовый минимум

Реализовать REST-style эндпойнты (create + get + list) для сущности users (хранить в памяти)


# Посложнее

Поддержать `PUT /users/{id}` для обновления пользователя, и `PATCH` для частичного (например, только пару полей) 


# Роскошный максимум

Поддержать `ETag` плюс `If-Match` заголовки для оптимистической конкуренции

 * В ответе возвращать ETag с номером версии user
 * Перед PUT, PATCH, DELETE проверять, совпадает ли заголовок ответа If-Match с номером версии user
 * Правильно отвечать 412 и 428
 * Не забыть про mutex lock
