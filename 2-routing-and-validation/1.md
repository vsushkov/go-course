# Small tasks

## 1: Routing patterns

* Реализовать хендлер для `GET /users/{id}` и безопасно получить из пути запроса {id}
* Возвращать 400 код ответа для невалидных ID, 404 для ненайденных


## 2: Validation helpers

* Написать небольшую функцию (функции) для валидации (например, `validateEmail`, `validateLen`, `validateRequired`)
* Использовать их для тела запроса в хендлере `POST /users`


## 3: Error envelope

* Стандартизировать все ошибки: `{"error":{"code":"...","message":"...","request_id":"..."}}`
* Убедиться, что все хендлеры возвращают ошибки только в таком виде
