# Small tasks

## Эндпойнт для получения JWT токена

* Написать эндпойнт для POST /auth/login (пользователя можно захардкодить, не обязательно брать из БД). В ответе должен
  возвращаться JWT токен с claims "sub" и "exp"


## Middleware для аутентификации

* В middleware верифицировать JWT, не забыть проверить случаи
  * токена нет в заголовке Authorization
  * токен невалидный
  * токен просрочен
* Если с токеном все хорошо, помещать ID пользователя в context запроса


## Проверка прав

* Добавить claim "role". Реализовать middleware/helper `RequireRole("admin")`
* Закрыть `DELETE /notes/{id}` ото всех, кроме владельцев роли admin.
