# Big task

## Базовый минимум

Добавь асинхронную "индексацию": при создании note / task, помещать задачу в очередь для обработки текста (подсчёт
количества слов и запись результата в отдельную колонку в таблице).


## Посложнее

К базовому минимуму добавить:
1. Ограничить размер очереди задач на индексацию 
2. Если очередь заполнена, отвечать клиенту 503 Service Unavailable (поможет unblocking send c select)


## Роскошный максимум

К Посложнее добавить graceful shutdown. Если отправить SIGTERM (нажать Ctrl+C), то программа должна:
1. Перестать принимать новые HTTP запросы
2. Перестать принимать новые задачи на индексацию
3. Дождаться, пока все воркеры закончат свою работу с текущими задачами
4. Супер-роскошный мксимум: ограничить таймаутом, сколько у воркеров есть время на звершение

Постараться сделать не через context.Context, а на своих каналах. WaitGroup использовать - ок.
