# Big task

## Базовый минимум

Написать один интеграционный тест на авторизацию + создание note + получение note.
При этом, слой базы данных в тест не включать, заменить репозиторий fake-ом (in-memory).

## Посложнее

Тестирование gRPC с применением [bufconn](https://pkg.go.dev/google.golang.org/grpc/test/bufconn).
Необходимо написать один интеграционный тест, проверив создание + получение note. Слой БД исключить из теста, 
заменив репозиторий fake-ом. Не забыть проверить статус ответа.


## Роскошный максимум

Написать интеграционный тест, чтобы удостовериться в том, что код конкуренто-безопасный, и не возникает data races.
Пусть будет происходить несколько параллельных записей в репозиторий (in-memory, не БД). 
Убедиться, что
 - нет data races (запускать тест с `go test -race`)
 - получаются корректные результаты: было создано ровно N сущностей, все ID уникальные, ничего не потерялось и не задублировалось
Тестировать репозиторий/сервис. Чтобы тест был качественным, надо раз 200 конкурентно вызвать repo.CreateNote()
